## Description

When `strictNullChecks` is enabled, TypeScript infers `candles[idx - 1]` as
non-nullable because `Candle[]` elements are non-null. Using optional chaining
(`candles[idx - 1]?.close`) with a `??` fallback triggers TS2869:

```
TS2869: Right operand of ?? is unreachable because the left operand is never nullish.
```

The fix is to add an explicit guard (`if (idx < 1) break;`) before accessing
`candles[idx - 1]`, then access directly without optional chaining.

Occurred twice in this session:
1. ATR percentile loop in `computeRiskParams()`
2. `computeAtrSync()` function

## Evidence

- Session epic-9-v2-confluence: In `confluence.service.ts`, both `computeRiskParams()`
  and `computeAtrSync()` had `candles[idx - 1]?.close ?? candles[idx].close`.
  Produced TS2869 in production build. Fixed by adding `if (idx < 1) break;`
  guard and accessing `candles[idx - 1].close` directly.

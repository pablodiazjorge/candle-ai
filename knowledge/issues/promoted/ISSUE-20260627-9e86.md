## Description

`@ngx-translate/core` has a static initializer in `fesm2022/ngx-translate-core.mjs` that imports `DefaultMissingTranslationHandler`, which triggers Angular JIT compilation when running vitest. Simply providing a mock `TranslateService` via `injectMocks.set()` is NOT sufficient — the static initializer runs at module load time. Must use `vi.mock('@ngx-translate/core', ...)` at the TOP of the test file, before any imports.

## Evidence

- Session 2026-06-27-i18n: `confluence.service.spec.ts` failed with "JIT compilation failed for injectable [class DefaultMissingTranslationHandler]" — fixed by adding `vi.mock('@ngx-translate/core', ...)` with full module mock
- Session 2026-06-27-i18n: `analysis.service.spec.ts` failed with same error after adding `TranslateService` injection — fixed same way

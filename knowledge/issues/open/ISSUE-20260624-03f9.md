## Description

VS Code Copilot Chat has two settings that control instruction loading:
- **Chat: Include Applying Instructions** — includes instructions whose `applyTo` matches the current file
- **Chat: Include Referenced Instructions** — includes the actual CONTENT of referenced `.instructions.md` files

When "Include Referenced Instructions" is OFF, Copilot Chat only registers file references (e.g., `<file>default.instructions.md</file>`) but does NOT inject their content into the system prompt. The model sees the reference but not the text, so it must proactively `read_file` to get the content. With it ON, the full file content is injected.

This is separate from the `github.copilot.chat.instructions` VS Code setting which injects arbitrary text.

## Evidence

- Session 68e8a9c8: User noticed `default.instructions.md` startup sequence was not being followed. Investigation revealed the "Include Referenced Instructions" checkbox was unchecked in VS Code settings > Chat > Context. Solution: enable the checkbox.

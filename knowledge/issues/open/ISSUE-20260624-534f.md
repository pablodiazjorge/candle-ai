## Description

In Angular component templates, TypeScript class getters are accessed **without** parentheses (e.g., `bullishPatterns.length`), while `input()` signal properties require `()` to read their value (e.g., `patterns().length`). Confusing the two causes template compilation errors.

Error: `TS6234: This expression is not callable because it is a 'get' accessor. Did you mean to use it without '()'?`

## Evidence

- Session 68e8a9c8: In `pattern-overlay.html`, getters `bullishPatterns()`, `bearishPatterns()`, `neutralPatterns()` were called with `()` causing TS6234. Fixed by removing `()` from getter accesses.

## Description

When `noPropertyAccessFromIndexSignature` is enabled in `tsconfig.json` (Angular default strict mode), accessing properties on `Record<string, unknown>` or `JSON.parse()` results requires bracket notation: `obj['key']` instead of `obj.key`.

Error: `TS4111: Property 'key' comes from an index signature, so it must be accessed with ['key'].`

This is common when parsing LLM JSON responses where the shape is unknown at compile time.

## Evidence

- Session 68e8a9c8 (Phase 4): In `analysis.service.ts`, `parsed.levels.map(l => ({ price: l.price }))` failed with TS4111. Fixed with `l['price']`, `l['type']`, `l['strength']`, `parsed.trend?.['direction']`, `parsed.risk?.['score']`, etc.
